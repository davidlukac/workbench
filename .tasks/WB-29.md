Title: Refactor disk operations into a repository layer

Description: Extract all disk I/O operations (reading/writing files, creating directories, etc.) into a dedicated repository layer. This will improve testability, maintainability, and allow for easier switching between different storage backends in the future.

Acceptance Criteria:
- [ ] All file operations are moved to a new repository layer
- [ ] The repository layer is injectable and can be mocked for testing
- [ ] Existing functionality remains unchanged
- [ ] 100% test coverage maintained
- [ ] Prepare the interface in a way that another (sqlite) repository can replace the JSON store without changing business logic
- [ ] update memories, documentation and design gaps
- [ ] create multi-channel abstraction, so we can register multiple repositories (sqlite, JSON file), so write operations would broadcast to all channels, read from primary

Status: done
