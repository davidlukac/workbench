Title: MCP resources
Description: Implement all read-only workbench:// MCP resources. Resources must never mutate state.

Resources:
- workbench://story — current story
- workbench://spec — current spec
- workbench://tasks — full task list
- workbench://tasks/pending — claimable tasks (deps satisfied, unlocked)
- workbench://tasks/ready_for_signoff — tasks awaiting human approval
- workbench://tasks/{id} — single task + evidence
- workbench://personas — resolved persona definitions
- workbench://skills — skills mapped per persona

Acceptance Criteria:
- `resources/skills/workbench-emulator/SKILL.md` updated to guide the emulator to prefer reading story/spec/task/persona/skill state via `workbench://` MCP resources over reading emulator state files directly, with file-based fallback when unavailable.
- 100% test coverage maintained (`npm run test:coverage` passes).
- All `workbench://` resources are documented in the MCP server reference.
