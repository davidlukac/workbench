Title: Story status MCP tool
Status: done
Description: Implement a `update_story_status` MCP tool that persists story status transitions in the ledger. Args: story_id, status. Valid transitions follow the story FSM: `todo → in_progress → in_review → done`. The tool should also update the `source_file` on disk if one is set (same file-sync behaviour the emulator currently does manually).

Acceptance Criteria:
- `update_story_status` tool is implemented and registered in the MCP server. Args: story_id, status.
- Tool enforces valid FSM transitions and rejects invalid ones with a descriptive error.
- If the story has a `source_file` set, the tool updates the `status` field in that file (YAML frontmatter or `Status:` line for plain Markdown).
- `resources/skills/workbench-emulator/SKILL.md` updated to prefer calling `update_story_status` MCP tool for story status transitions, with file-based fallback when unavailable.
- 100% test coverage maintained (`npm run test:coverage` passes).
- `update_story_status` tool is documented in the MCP server reference.
